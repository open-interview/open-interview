> **Understanding Design a global, low-latency caching layer for a real-time quantitative trading platform handling tick data and signals across regions. Aim for 50 ms latency for hot instrument queries, support per-instrument hot/cold data with TTL-based eviction, cross-region invalidation, and automatic failover. Allocate Redis for hot caches, Memcached for legacy services, and use a CDN for static dashboards and precomputed signals. Include warming, privacy, and ops considerations?**

## Overview

Multi-tier cache architecture: regional Redis clusters for hot instrument data with sub-10ms access times, Memcached layer for legacy services, and CDN edge caching for static dashboards and precomputed signals. Implement cache-aside pattern with write-through for critical trading signals, utilizing Redis pub/sub for cross-region invalidation and automatic failover mechanisms. Deploy regional hot caches for active instruments (top 5% driving 70% of traffic) with TTL-based eviction policies, complemented by cold data persistence in base storage.
