> **Understanding You're releasing a critical change to a service with a hybrid automation stack: 5 on-prem Puppet agents, 3 AWS instances managed by Ansible, and a Kubernetes deployment controlled via Chef. Describe a concrete end-to-end zero-downtime rollout using blue/green, how you would enforce policy-as-code to block non-approved versions, and how you'd detect and recover from drift across Puppet, Ansible, and Chef. Include the exact artifacts you'd produce (manifests, playbooks, cookbooks, tests) and the commands to run?**

## Overview

Coordinate a blue/green rollout via a gating repository that tags traffic shifts. Puppet enforces approved versions on on-prem nodes using a version-locked environment.conf and r10k-controlled environments; Ansible deploys to AWS with a canary subset using the ansible-check mode and delegate_to to shift traffic incrementally; Chef Infra applies Kubernetes manifests via Chef InSpec for compliance validation before applying changes. Use OPA (Open Policy Agent) as the policy-as-code gate to validate all version constraints before promotion between environments.
