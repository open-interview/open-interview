> **Understanding Design a rate limiting system for a video streaming platform that needs to enforce different limits based on content type and user tier: free users can stream 480p video (5GB/month), premium users get 1080p (20GB/month), and 4K streaming (50GB/month) for enterprise. How would you implement bandwidth-based rate limiting that dynamically adjusts quality and prevents abuse while ensuring smooth playback experience?**

## Overview

Implement a token bucket system with bandwidth tokens instead of request tokens, using Redis for distributed state management and adaptive bitrate streaming to dynamically adjust quality based on remaining quota and network conditions.
