> **Understanding Channel: backend/authentication. Intermediate. You're building a multi-tenant API gateway that accepts OIDC ID tokens and SAML 2.0 assertions to mint per-tenant, short-lived internal JWTs. A new requirement is nonce-based replay protection. Describe a concrete plan to: (1) validate OIDC and SAML tokens and map to internal claims including tenant_id, user_id, and scopes; (2) cache per-tenant JWKS and SAML metadata with rotation policy; (3) enforce per-tenant audience and issuer; (4) implement a scalable distributed nonce store with TTL to prevent replay; (5) expose a /validate endpoint that returns effective claims for a token without full decoding, and show a minimal code example for issuing a new JWT with a nonce and for nonce verification during validation?**

## Overview

Validate OIDC and SAML tokens, extract tenant_id, user_id, and scopes; cache per-tenant JWKS and SAML metadata with automatic rotation; enforce per-tenant audience (aud) and issuer (iss) validation; implement a distributed nonce store with key format (tenant_id:nonce) and TTL for replay protection; expose a lightweight /validate endpoint that returns effective claims without full token decoding.
