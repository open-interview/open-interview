Include a concrete API sketch and a short code example?**

## Overview

I'd implement a modular architecture that fetches initial widget data via REST API, establishes a WebSocket connection for live updates, and leverages a Service Worker to cache assets and the latest payload for offline functionality. The solution includes: (1) a minimal in-browser patch-based DOM updater that efficiently applies incremental changes to widgets, (2) a WebSocket reconnect strategy with exponential backoff for resilient real-time communication, (3) a fetch wrapper that deduplicates concurrent in-flight requests and caches responses for 60 seconds to optimize performance, and (4) cross-tab state consistency using BroadcastChannel to ensure synchronized dashboard states across browser tabs.
