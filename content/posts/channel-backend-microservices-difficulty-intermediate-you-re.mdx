> **Understanding Channel: backend/microservices. Difficulty: intermediate. You're implementing a multi-tenant content pipeline where an article flows through encoding, thumbnail generation, and CDN publishing, with per-tenant billing events. Using CQRS, Event Sourcing, and a Saga orchestrator, outline a concrete plan that ensures per-tenant isolation, exactly-once processing, and compensating actions if CDN publish fails after encoding and thumbnailing. Provide a minimal code sketch showing the orchestrator starting the Saga and the compensating handler?**

## Overview

Plan: tag every event with `tenantId` and `articleId`, orchestrating Encoding → Thumbnails → CDN via a per-tenant outbox. Ensure idempotence using a `StepState` keyed by `(tenantId, articleId, step)` and replay-safe handlers. If the CDN publish fails post-encoding and thumbnailing, trigger compensating actions: rollback the CDN write, then revert the thumbnail and encoded asset, finally emitting a `PipelineFailed` event with the compensation log for auditability.
