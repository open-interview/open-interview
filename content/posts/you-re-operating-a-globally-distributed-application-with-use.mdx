> **Understanding You're operating a globally distributed application with users across North America, Europe, and Asia. Each region has its own data center, but you occasionally see significant cross-region latency spikes when users in Europe get routed to US-East servers due to DNS misconfigurations. How would you design a multi-region load balancing strategy that optimizes for latency, handles data sovereignty requirements, and gracefully degrades when an entire region fails?**

## Overview

Combine GeoDNS with latency-based routing using real-time measurements, implement regional fallbacks with deterministic hashing to ensure data locality, and use active health checks with automatic failover.
