> **Understanding Design an idempotent, auditable secret rotation workflow across hybrid on-prem and cloud, using Ansible, Puppet, and Chef with Vault as the source of truth and AWS Secrets Manager integration. How would you implement, test, and monitor rotations end-to-end, including credentials propagation and failure handling?**

## Overview

Leverage Vault as the single source of truth for dynamic secrets. Ansible fetches secrets at runtime via the HashiCorp Vault lookup plugin, ensuring credentials are never hardcoded. Puppet manages rotation triggers and ensures dependent services reload when new credentials are available. Chef reconciles state across nodes, applying consistent rotation policies and validating successful propagation. Together, these tools provide idempotent, auditable secret rotation with built-in failure handling and rollback capabilities.
