> **Understanding Develop a multi-stage Dockerfile for a FastAPI service that compiles a C extension, runs as a non-root user, and fetches private dependencies using BuildKit secrets. Pair it with a docker-compose setup (dev/prod) including a Postgres DB and Redis, with proper secrets handling and secret rotation without image rebuild. Explain the build, run, and security trade-offs. How would you implement this end-to-end?**

## Overview

Use a multi-stage Dockerfile with a builder stage that leverages BuildKit's --mount=type=secret to fetch private dependencies securely. Install build tools and compile the C extension in this stage, then copy only the necessary artifacts to a slim final stage. Run the container as a non-root user with pinned dependency hashes for reproducibility. In docker-compose, mount runtime secrets from files (e.g., /run/secrets/) rather than embedding them in the image—this enables secret rotation without rebuilding the image.
