> **Understanding You're building an API platform that serves REST, GraphQL, and gRPC clients for a multi-tenant config service. Design a runtime contract governance model driven by a single OpenAPI spec, with protocol adapters, safe migrations, and per-tenant deprecation. Provide concrete mappings: REST GET /config/{env}, GraphQL type Config { env: String, settings: JSON }, query getConfig(env: ID!, version: String): Config, and gRPC ConfigService.GetConfig(GetConfigRequest) { string env; string version; }. Explain validation, testing, and observability considerations?**

## Overview

Use one OpenAPI contract as the source of truth. Expose REST, GraphQL, and gRPC adapters that map to a common Config entity. Implement runtime version negotiation via the X-Contract-Version header, with per-tenant deprecation timelines enforced by the adapters. Use a migration registry to track schema evolution, and validate all protocol payloads against the canonical OpenAPI spec at runtime.
