> **Understanding Design an offline-first, real-time collaborative whiteboard using a canvas element. The app should update strokes across clients via a WebSocket, cache the app shell and a snapshot of the board via a Service Worker for offline use, render efficiently with a single canvas and requestAnimationFrame, and reconcile edits from multiple clients with a simple CRDT-like op-Log. Include skeletons for: WebSocket wrapper (connect, backoff, send, onMessage), op-merge, Service Worker install/activate/cache, and main render wiring?**

## Overview

Use a canvas-centric architecture with a single rendering loop powered by requestAnimationFrame. The WebSocket wrapper batches stroke events into an operation log and sends deltas to clients. A CRDT-like merge function reconciles remote operations, which are then applied through the render loop to update the canvas efficiently.
