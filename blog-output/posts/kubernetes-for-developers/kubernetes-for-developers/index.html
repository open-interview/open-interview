Kubernetes for Developers

Kubernetes has become the de facto standard for container orchestration. As a developer, understanding Kubernetes helps you ship more reliable software and collaborate better with your ops team.

Core Concepts

  • Pods — the smallest deployable unit, wrapping one or more containers
  • Deployments — manage pod replicas and rolling updates
  • Services — expose pods to network traffic with stable DNS names
  • ConfigMaps & Secrets — externalize configuration from your containers
  • Ingress — route external HTTP traffic to services

Your First Deployment

apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: my-app:1.0.0
        ports:
        - containerPort: 8080
        resources:
          requests:
            memory: "64Mi"
            cpu: "250m"
          limits:
            memory: "128Mi"
            cpu: "500m"

Health Checks

Always configure liveness and readiness probes:

livenessProbe:
  httpGet:
    path: /health
    port: 8080
  initialDelaySeconds: 30
  periodSeconds: 10
readinessProbe:
  httpGet:
    path: /ready
    port: 8080
  initialDelaySeconds: 5
  periodSeconds: 5

Developer Tips

  • Use kubectl port-forward to access services locally during development
  • kubectl logs -f <pod> streams logs in real time
  • kubectl exec -it <pod> -- /bin/sh opens a shell inside a running container
