Study Strategy

  1. Understand the Well-Architected Framework — the exam is built around its five pillars
  2. Practice with hands-on labs — theory alone won't cut it
  3. Focus on high-frequency services — EC2, S3, RDS, VPC, IAM, Lambda
  4. Take practice exams — aim for 85%+ before scheduling the real thing

Key Services to Master

Compute

  • EC2 — instance types, auto scaling groups, placement groups
  • Lambda — serverless functions, triggers, concurrency limits
  • ECS/EKS — container orchestration options

Storage

  • S3 — storage classes, lifecycle policies, versioning, cross-region replication
  • EBS — volume types (gp3, io2), snapshots, encryption
  • EFS — shared file storage for Linux workloads

Networking

  • VPC — subnets, route tables, security groups, NACLs
  • Route 53 — DNS routing policies (weighted, latency, failover)
  • CloudFront — CDN, edge locations, cache behaviors

Common Exam Traps

  • Multi-AZ vs. Read Replicas — Multi-AZ is for high availability, Read Replicas are for read scaling
  • Security Groups vs. NACLs — SGs are stateful, NACLs are stateless
  • S3 consistency — S3 now provides strong read-after-write consistency for all operations
