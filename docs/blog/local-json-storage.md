# Blog Local JSON Storage

## Overview

The blog system runs without PostgreSQL by using local JSON files as the data source. This enables the site to work on static hosts (GitHub Pages, Replit free tier) with zero database setup.

- **Source of truth**: `data/blog-posts.json` (121 posts)
- **Static build output**: `client/public/data/posts.json`
- **Storage adapter**: `server/blog-storage.ts` (uses `STATIC_BUILD` fallback)
- **Fallback chain**: PostgreSQL → `client/public/data/posts.json` → empty array

When `STATIC_BUILD=true` is set (or the DB is unreachable), the server reads all posts from the static JSON file instead of querying PostgreSQL.

## Architecture

```
Blog API (server/routes.ts)
└── blogStorage (server/blog-storage.ts)
    └── STATIC_BUILD mode
        └── client/public/data/posts.json (served statically)
            ↑ generated from
        └── data/blog-posts.json (source of truth)
```

The conversion step (`scripts/convert-blog-to-posts-json.js`) transforms `data/blog-posts.json` into the format expected by `BlogPost` interface and writes it to `client/public/data/posts.json`.

## Data Flow: JSON Fields → BlogPost Interface

`data/blog-posts.json` uses a different field naming convention than the `BlogPost` interface. The conversion maps fields as follows:

| `data/blog-posts.json` field | `BlogPost` interface field | Notes |
|------------------------------|---------------------------|-------|
| `id` | `id` | Unique post identifier |
| `blogTitle` | `title` | Post title |
| `blogSlug` | `slug` | URL slug |
| `blogIntro` | `excerpt` | Short summary |
| `blogSections[].heading` + `blogSections[].content` | `content` | Joined into full content string |
| `blogConclusion` | `content` (appended) | Appended to content |
| `channel` | `category` | Post category/channel |
| `tags` | `tags` | Array of tag strings |
| `sources` | *(not in interface)* | Discarded during conversion |
| `difficulty` | *(not in interface)* | Discarded during conversion |
| *(derived)* | `author` | Always `"TechExpert AI"` |
| *(derived)* | `readingTimeMinutes` | Estimated from content length |
| *(derived)* | `publishedAt` | Set from post metadata |
| *(derived)* | `featured` | Defaults to `false` |
| *(derived)* | `coverImage` | `undefined` (no image in source) |
| *(derived)* | `status` | Defaults to `"published"` |

## Setup

No setup required. The JSON files are committed to the repository:

- `data/blog-posts.json` — source data (121 posts)
- `client/public/data/posts.json` — static build output (generated by conversion script)
- `client/public/data/` — also contains `channels.json` and `tags` lists

The server automatically detects whether to use PostgreSQL or JSON fallback based on the `DATABASE_URL` environment variable and `STATIC_BUILD` flag.

## Adding Posts

### Option 1: Edit `data/blog-posts.json` directly

Add a new object to the root array following the existing format:

```json
{
  "id": "blog-<timestamp>-<random>",
  "blogTitle": "Your Post Title",
  "blogSlug": "your-post-slug",
  "blogIntro": "Brief introduction...",
  "blogSections": [
    {
      "heading": "Section Heading",
      "content": "Section content..."
    }
  ],
  "blogConclusion": "Conclusion text...",
  "blogMeta": "Meta description",
  "channel": "backend",
  "tags": ["tag1", "tag2"],
  "sources": []
}
```

Then regenerate the static build:

```bash
node scripts/convert-blog-to-posts-json.js
```

### Option 2: Add MDX to `content/posts/` and convert

Place a new `.mdx` file in `content/posts/` following the existing MDX format, then run:

```bash
node scripts/build-content.ts
```

This will convert MDX posts to JSON and update `client/public/data/posts.json`.

## Troubleshooting

### 404 on blog pages
Check that the data files exist:
```bash
ls -la data/blog-posts.json
ls -la client/public/data/posts.json
```
Verify the post slug exists in the JSON array.

### Empty blog / no posts showing
Check JSON format is valid:
```bash
jq 'length' data/blog-posts.json
jq '.[0]' data/blog-posts.json
```
Common issues: trailing commas, missing `blogSlug` field, or empty array.

### Search not working
Verify `tags` is an array of strings (not objects or comma-separated strings):
```bash
jq '.[0].tags' data/blog-posts.json
```
Should output: `["tag1", "tag2"]` not `"tag1, tag2"`.

### Posts not updating on site
The static JSON (`client/public/data/posts.json`) may be out of sync with the source. Re-run the conversion:
```bash
node scripts/convert-blog-to-posts-json.js
```

## Performance

- **JSON loaded into memory** on server startup (via `loadStaticData()` in `server/blog-storage.ts`)
- **No database queries** in `STATIC_BUILD` mode
- **Sub-millisecond response times** for post lookups (in-memory array filtering)
- **Static files** (`posts.json`) are cacheable by CDNs and browsers

The trade-off: all 121 posts are loaded into memory on each request in `STATIC_BUILD` mode (the `loadStaticData()` function is called per-request, but reads from the filesystem). For high-traffic scenarios, add a simple in-memory cache:

```typescript
let cachedData: { posts: BlogPost[]; categories: BlogCategory[]; tags: string[] } | null = null;
function loadStaticData() {
  if (cachedData) return cachedData;
  // ... load from file
  cachedData = result;
  return result;
}
```
