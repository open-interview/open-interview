# Open Interview вЂ” Astro Blog

State-of-the-art blog powered by [Astro](https://astro.build) + Tailwind CSS.
Deployed to GitHub Pages at [open-interview.github.io](https://open-interview.github.io).

## Features

- **121 posts** exported from `client/public/blog-data.json` (clean markdown, mermaid diagrams, Key Takeaways)
- Dark mode by default with Slate/Blue palette
- **Mermaid diagrams** rendered client-side with dark theme
- **Syntax highlighting** via Shiki (`github-dark` theme)
- **Reading progress bar** on every post
- **TOC sidebar** with active section highlighting
- **Tag + Category filtering** on the index page (client-side, zero JS framework)
- **RSS feed** at `/rss.xml`
- **Sitemap** at `/sitemap-index.xml`
- Prev/Next post navigation
- Copy-to-clipboard code blocks
- Lighthouse 100/100/100/100 profile

# Then run the Astro dev server

```bash
cd blog-astro

```

## Deployment

The GitHub Actions workflow `.github/workflows/deploy-astro-blog.yml` runs automatically when:
- `client/public/blog-data.json` changes (after a content pipeline run)
- `blog-astro/**` files change
- Manually triggered via `workflow_dispatch`

It:
1. Exports `blog-data.json` в†’ `src/content/blog/*.md`
2. Runs `astro build`
3. Deploys `dist/` to GitHub Pages via `actions/deploy-pages@v4`

## Content Schema

Posts come from `client/public/blog-data.json` via the export script. Frontmatter fields:

| Field | Type | Description |
|-------|------|-------------|
| `title` | string | Post title |
| `excerpt` | string | Short description |
| `category` | string | e.g. `system-design`, `devops` |
| `tags` | string[] | e.g. `["rate-limiting", "api"]` |
| `publishedAt` | date | ISO 8601 |
| `difficulty` | `beginner` \| `intermediate` \| `advanced` | |
| `readingTimeMinutes` | number | |
| `featured` | boolean | Shows in featured row |
| `author` | string | |
