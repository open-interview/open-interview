# Goals-Signals-Metrics (GSM) Framework
*Based on Google UX Patterns Section 13.2*

## Top 5 Key Features

### 1. Practice (Standard & Extreme Modes)
- **Goal**: Help users prepare for technical interviews by practicing question sets with immediate feedback.
- **Signal**: Users complete practice sessions, answer questions correctly, return for subsequent sessions.
- **Metric**: Session completion rate, average accuracy rate, weekly active practice users (WAPU), average session duration.

### 2. Voice Practice (VoiceSessions)
- **Goal**: Enable users to simulate voice-based interview scenarios to improve verbal communication skills.
- **Signal**: Users complete full voice sessions, use voice commands to navigate, replay voice feedback.
- **Metric**: Voice session completion rate, average session duration, voice recognition accuracy rate, weekly active voice users.

### 3. Flashcards (Spaced Repetition System)
- **Goal**: Improve long-term retention of technical concepts via spaced repetition flashcards.
- **Signal**: Users review flashcards daily, mark cards as "known", progress through decks.
- **Metric**: Daily active flashcard users, average cards reviewed per user, retention rate (cards marked known after 7+ days).

### 4. Tests (Timed Exam Simulations)
- **Goal**: Assess user readiness for real technical interviews via timed, proctored test environments.
- **Signal**: Users complete full tests, pass with required score, retake tests to improve.
- **Metric**: Test completion rate, pass rate, average test score, retake rate.

### 5. Learning Paths (Structured Roadmaps)
- **Goal**: Guide users through structured learning journeys to achieve certification or skill mastery.
- **Signal**: Users enroll in paths, complete milestones, finish full learning paths.
- **Metric**: Path enrollment rate, milestone completion rate, path completion rate, average time to complete path.
